### Welcome to the Praetorian Guard Platform (PGP)!

This page offers step-by-step instructions to sign up and get started in PGP.

## Sign up

1\. Navigate to https://PGP.praetorian.com/login 2. On your first visit, you'll see several options:- Sign in with email/password- Sign in with Single Sign-On (SSO)- Sign in with Google- Sign up (for new users)

Enter your email and create a password on the **Sign Up** page. You'll receive a verification code via email within a few minutes to confirm your address.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-e005-734a-9e18-f6c5d21eba68/b64u-MDE5YjdmZjctZGVlYy03Y2U0LTk5YzctZjA4NDM1MWM3ZmQw.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=a968f47602a766c4269c20cfa71daf119afdccce21213ee00f562462b2fffaa6)

When you click **Continue** you will be prompted for the code that was sent to the email you used to **Sign Up**.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-df4d-7f1d-9c9d-2f14d3e96862/b64u-MDE5YjdmZjctZGU1Ni03NzBhLThiYjAtYTM0YWQ0M2Y1MjY5.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=c35b2057a8b415a88010c651f2c3bd96120236b06f4e84ad0d5207836f8ebf84)

Your verification email will come from PGP-noreply@praetorian.com with the subject "Your verification code".

When you enter the code into the boxes provided, you will be authenticated into PGP.

This marks the beginning of your PGP journey!

Upon accessing PGP for the first time, you will be presented with the Metrics dashboard:

## Seeds

To start PGP scans, navigate to the Seeds page by clicking **Seeds** on the navigation bar on the left side of the screen.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-e3e7-72a7-adb2-577807c7d2d8/b64u-MDE5YjdmZjctZTNjNi03YWIzLTkzYjMtN2ExNzU2NWQ4OTc5.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=0fb3ff743d3a108f32a19fce5d55697122ffc47540e0fe9833a9a5ed10e5e005)

In PGP, a seed is a persistent digital asset owned by your organization. You can input top level domains, fully qualified domain names, CIDR ranges, IP addresses, GitHub organizations, or GitHub repositories. These should be stable resources. In other words, an ephemeral IP address managed by a cloud service would not be a good addition as a seed. The domain that IP is tied to, would be a good addition as a seed.

On the top right of the Seeds table, there is an **Add Seed** button. Click the **Add Seed** button.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-e584-781c-ad7d-3e80e338f732/b64u-MDE5YjdmZjctZTRhNi03NGMzLWJkNDQtODliNDU0ZWNlMzRi.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=8322a52228328b45cac9b0f687532fc8ba3140362f0959e3b7228d03aa478218)

A pop-up will appear. In this pop-up, you can add an individual seed, or add a file with a list of seeds.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-e79b-79f6-9f1c-f5a50b8290e9/b64u-MDE5YjdmZjctZTczZS03MmFmLWFmNzUtODVkMzI2MjA1YWFm.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=5d6a8675512bb0014c31e11597b1286491c63e3bac066c7c050dd9a9451942c9)

You can input any of these as seeds:

\- Top-level domains (e.g., example.com)- Fully qualified domain names (e.g., subdomain.example.com)- CIDR ranges (e.g., 192.168.1.0/24)- IP addresses (e.g., 192.168.1.1)- GitHub organizations (https://github.com/praetorian-inc)- GitHub repositories (https://github.com/praetorian-inc/noseyparker)

Once you input a seed, another pop-up will ask which type of scanning you want PGP to perform:

Choose one of two scanning options:- **Enumerate Assets Only**: Identifies all digital assets associated with your seed - **Discover Vulnerabilities**: First identifies assets, then performs security scans on each one

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-e85c-7b2f-b994-65a97ab83c72/b64u-MDE5YjdmZjctZTg0Yy03YTU5LWE2Y2MtOGE1YmJjZjNmMzRm.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=f043105ca36daa75d4695240bba387d2060093bd25483f1cbc72687823fc2e9f)

## Assets

After you've added your root domain and other seeds, click **Assets** on the navigation bar on the left side of the screen.

The **Assets** page is where you can see your full attack surface in PGP.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-eb30-785f-b1ed-4ec484f23863/b64u-MDE5YjdmZjctZWE1ZC03OWZmLWI1ZjAtZTlkYzU3Nzg1MzZm.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=654b90878ece61733ae7a0b7393b41f60cad5fbc93e6f98c8fc284e997400075)

You can click on any asset to see more information.

Each asset will have an **Overview**, **Vulnerabilities** (if any are found), **Domain** information, **Technologies** associated with the asset, **Cloud** information, and a section to add **Notes**.

## Users

PGP allows cross-account collaboration. By adding a collaborator, you are enabling another PGP user access to your account. This is useful when setting up an organization account with employees as collaborators. For detailed instructions on collaborator management, see this documentation. When authorized users and collaborators are set, the **Users** page is where you can manage access.

## Integrations

To get the most from PGP, we recommend integrating with your existing service providers. PGP offers integrations to applications that provide **Breach and Attack Simulation, Cloud Security Posture Management, Cloud Service Providers, Content Delivery Solutions, Cyber Asset Attack Surface Management, Firewall, IT Service Management, Managed Detection and Response, Managed DNS, Passive DNS, Source Code Management, and Vulnerability Management** services.

From the **Integrations** page, you can connect PGP and your existing services. Documentation for each integration can be found in the Integrations section.

## Settings

The **Settings** page provides comprehensive control over your PGP account configuration, notifications, security features, and appearance preferences.

![](https://69448200be4d4ffad9065e4d.featurebase-attachments.com/c/article/69580e54a21178a432dd63e2/019b7ff7-f839-7e14-960e-10060288dba2/b64u-MDE5YjdmZjctZjcxNi03ZTkwLWJmMDQtYjQxODI2MmJlMzBj.png?X-Amz-Expires=3600&X-Amz-Date=20260318T050000Z&X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=DO801TYC4FCVNNEKURKM%2F20260318%2Ffra1%2Fs3%2Faws4_request&X-Amz-SignedHeaders=host&X-Amz-Signature=81079d327b94c987224dd52084257b8b6993e7778ae1842d764ed4a3792acdec)

On the **Settings** page, you can manage:

\- **Organization Details**; View and edit your organization name and contact email.

\- **Notifications**; Configure alerts through external integration channels. Set up Exposure Alerts to monitor externally exposed ports and services. Note that Notifications must be configured before adding Exposure Alerts.

\- **Single Sign-On**; Enable and configure Single Sign-On (SSO) authentication for your organization's users.

\- **Webhook URL**; Generate a unique URL to push assets and risks to PGP.

\- **Whitelisting Details**; Each scan includes your account-unique header that can be added to your security tools' allowlist to prevent scanning disruptions.

\- **Interface Theme**; Customize your application appearance with system, dark, or light mode options.
